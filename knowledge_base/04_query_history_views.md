# Query & Session History Views

These views capture the full execution history of queries, login events, and user sessions in your Snowflake account. They are the primary source of truth for **performance monitoring**, **usage attribution**, **cost allocation**, and **security auditing**.

---

## QUERY_HISTORY

**Purpose:** Records metadata for every SQL statement executed in your account over the past 365 days. This is the most-used view in the ACCOUNT_USAGE schema.

**Data Latency:** 45 minutes to 3 hours  
**Retention:** 1 year (365 days)

**Key Columns:**

| Column | Type | Description |
|--------|------|-------------|
| `QUERY_ID` | TEXT | System-generated unique identifier for the query |
| `QUERY_TEXT` | TEXT | Full SQL text (up to 100K characters; truncated for longer queries) |
| `DATABASE_ID` | NUMBER | Internal ID of the database context |
| `DATABASE_NAME` | TEXT | Database set in the session when the query ran |
| `SCHEMA_ID` | NUMBER | Internal ID of the schema context |
| `SCHEMA_NAME` | TEXT | Schema set in the session when the query ran |
| `QUERY_TYPE` | TEXT | Type of query: SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, MERGE, COPY, etc. |
| `SESSION_ID` | NUMBER | Session identifier |
| `USER_NAME` | TEXT | Snowflake user who ran the query |
| `ROLE_NAME` | TEXT | Active role during query execution |
| `WAREHOUSE_ID` | NUMBER | Internal ID of the warehouse used |
| `WAREHOUSE_NAME` | TEXT | Name of the virtual warehouse |
| `WAREHOUSE_SIZE` | TEXT | Warehouse size (X-Small, Small, Medium, Large, etc.) |
| `WAREHOUSE_TYPE` | TEXT | Warehouse type (STANDARD, SNOWPARK-OPTIMIZED) |
| `CLUSTER_NUMBER` | NUMBER | Cluster number if multi-cluster warehouse |
| `QUERY_TAG` | TEXT | Query tag set in the session or via ALTER SESSION |
| `EXECUTION_STATUS` | TEXT | Status: SUCCESS, FAIL, INCIDENT |
| `ERROR_CODE` | TEXT | Error code if query failed |
| `ERROR_MESSAGE` | TEXT | Error message if query failed |
| `START_TIME` | TIMESTAMP_LTZ | Timestamp when query execution started |
| `END_TIME` | TIMESTAMP_LTZ | Timestamp when query execution completed |
| `TOTAL_ELAPSED_TIME` | NUMBER | Total time from start to end (milliseconds) |
| `BYTES_SCANNED` | NUMBER | Bytes scanned from storage |
| `PERCENTAGE_SCANNED_FROM_CACHE` | FLOAT | Fraction of bytes served from the result cache |
| `BYTES_WRITTEN` | NUMBER | Bytes written to storage |
| `BYTES_WRITTEN_TO_RESULT` | NUMBER | Bytes written to the result set |
| `BYTES_READ_FROM_RESULT` | NUMBER | Bytes read from cached results |
| `ROWS_PRODUCED` | NUMBER | Number of rows returned |
| `ROWS_INSERTED` | NUMBER | Rows inserted (DML) |
| `ROWS_UPDATED` | NUMBER | Rows updated (DML) |
| `ROWS_DELETED` | NUMBER | Rows deleted (DML) |
| `ROWS_UNLOADED` | NUMBER | Rows unloaded (COPY INTO stage) |
| `BYTES_DELETED` | NUMBER | Bytes deleted |
| `PARTITIONS_SCANNED` | NUMBER | Partitions scanned |
| `PARTITIONS_TOTAL` | NUMBER | Total partitions accessed |
| `BYTES_SPILLED_TO_LOCAL_STORAGE` | NUMBER | Bytes spilled to local disk (indicates memory pressure) |
| `BYTES_SPILLED_TO_REMOTE_STORAGE` | NUMBER | Bytes spilled to remote disk (severe memory pressure) |
| `BYTES_SENT_OVER_THE_NETWORK` | NUMBER | Network bytes sent |
| `COMPILATION_TIME` | NUMBER | Time spent compiling the query (ms) |
| `EXECUTION_TIME` | NUMBER | Time spent executing the query (ms) |
| `QUEUED_PROVISIONING_TIME` | NUMBER | Time waiting for warehouse provisioning (ms) |
| `QUEUED_REPAIR_TIME` | NUMBER | Time waiting for warehouse repair (ms) |
| `QUEUED_OVERLOAD_TIME` | NUMBER | Time queued due to warehouse overload (ms) |
| `TRANSACTION_BLOCKED_TIME` | NUMBER | Time blocked by transaction locks (ms) |
| `OUTBOUND_DATA_TRANSFER_CLOUD` | TEXT | Target cloud for outbound data transfer |
| `OUTBOUND_DATA_TRANSFER_REGION` | TEXT | Target region for outbound data transfer |
| `OUTBOUND_DATA_TRANSFER_BYTES` | NUMBER | Bytes transferred outbound |
| `INBOUND_DATA_TRANSFER_CLOUD` | TEXT | Source cloud for inbound data transfer |
| `INBOUND_DATA_TRANSFER_REGION` | TEXT | Source region for inbound data transfer |
| `INBOUND_DATA_TRANSFER_BYTES` | NUMBER | Bytes transferred inbound |
| `CREDITS_USED_CLOUD_SERVICES` | FLOAT | Cloud services credits consumed |
| `IS_CLIENT_GENERATED_STATEMENT` | BOOLEAN | Whether the query was generated by the Snowflake client |
| `QUERY_ACCELERATION_BYTES_SCANNED` | NUMBER | Bytes scanned using Query Acceleration Service |
| `QUERY_ACCELERATION_PARTITIONS_SCANNED` | NUMBER | Partitions scanned by QAS |
| `QUERY_ACCELERATION_UPPER_LIMIT_SCALE_FACTOR` | NUMBER | QAS scale factor cap |

**Sample Queries:**
```sql
-- Top 10 most expensive queries by execution time (last 7 days)
SELECT query_id, user_name, warehouse_name, query_type,
       ROUND(total_elapsed_time / 1000, 2) AS elapsed_seconds,
       ROUND(bytes_scanned / 1024 / 1024 / 1024, 2) AS gb_scanned,
       execution_status
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -7, CURRENT_TIMESTAMP())
ORDER BY total_elapsed_time DESC
LIMIT 10;

-- Daily query volume and average execution time by warehouse
SELECT DATE(start_time) AS query_date,
       warehouse_name,
       COUNT(*) AS query_count,
       ROUND(AVG(total_elapsed_time) / 1000, 2) AS avg_elapsed_sec,
       SUM(CASE WHEN execution_status = 'FAIL' THEN 1 ELSE 0 END) AS failed_queries
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_TIMESTAMP())
GROUP BY 1, 2
ORDER BY 1 DESC, 2;

-- Find queries with high data spill (memory pressure indicators)
SELECT query_id, user_name, warehouse_name,
       ROUND(bytes_spilled_to_local_storage / 1024 / 1024 / 1024, 2) AS local_spill_gb,
       ROUND(bytes_spilled_to_remote_storage / 1024 / 1024 / 1024, 2) AS remote_spill_gb,
       warehouse_size
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -7, CURRENT_TIMESTAMP())
  AND (bytes_spilled_to_local_storage > 0 OR bytes_spilled_to_remote_storage > 0)
ORDER BY bytes_spilled_to_remote_storage DESC
LIMIT 20;

-- Cost by user (approximate, using warehouse credits)
SELECT user_name,
       COUNT(*) AS query_count,
       ROUND(SUM(total_elapsed_time) / 1000 / 3600, 2) AS total_hours,
       ROUND(SUM(credits_used_cloud_services), 4) AS cloud_services_credits
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_TIMESTAMP())
GROUP BY user_name
ORDER BY total_hours DESC;

-- Cache utilization rate
SELECT DATE(start_time) AS query_date,
       ROUND(AVG(percentage_scanned_from_cache) * 100, 2) AS avg_cache_hit_pct,
       COUNT(*) AS query_count
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -30, CURRENT_TIMESTAMP())
  AND execution_status = 'SUCCESS'
GROUP BY 1
ORDER BY 1 DESC;
```

## LOGIN_HISTORY

**Purpose:** Records every login attempt (successful and failed) to your Snowflake account. Critical for **security monitoring**, **audit compliance**, and detecting anomalous access patterns.

**Data Latency:** 2 hours  
**Retention:** 1 year

**Key Columns:**

| Column | Type | Description |
|--------|------|-------------|
| `EVENT_ID` | NUMBER | Unique identifier for the login event |
| `EVENT_TIMESTAMP` | TIMESTAMP_LTZ | Timestamp of the login attempt (UTC) |
| `EVENT_TYPE` | TEXT | Always "LOGIN" |
| `USER_NAME` | TEXT | Snowflake user attempting to log in |
| `CLIENT_IP` | TEXT | IP address of the connecting client |
| `REPORTED_CLIENT_TYPE` | TEXT | Client type (WEB, JDBC, ODBC, PYTHON, GO, etc.) |
| `REPORTED_CLIENT_VERSION` | TEXT | Version of the client software |
| `FIRST_AUTHENTICATION_FACTOR` | TEXT | Primary authentication method (PASSWORD, OAUTH, KEYPAIR, etc.) |
| `SECOND_AUTHENTICATION_FACTOR` | TEXT | MFA method used (DUO, NULL if no MFA) |
| `IS_SUCCESS` | TEXT | Login result: YES or NO |
| `ERROR_CODE` | NUMBER | Error code if login failed |
| `ERROR_MESSAGE` | TEXT | Error message if login failed |
| `RELATED_EVENT_ID` | NUMBER | Related event ID (links to SESSION_HISTORY) |
| `CONNECTION` | TEXT | Connection identifier |

**Sample Queries:**
```sql
-- Find all failed login attempts in the last 24 hours
SELECT event_timestamp, user_name, client_ip,
       reported_client_type, error_code, error_message
FROM SNOWFLAKE.ACCOUNT_USAGE.LOGIN_HISTORY
WHERE event_timestamp >= DATEADD('hour', -24, CURRENT_TIMESTAMP())
  AND is_success = 'NO'
ORDER BY event_timestamp DESC;

-- Detect brute-force patterns: users with many failed logins
SELECT user_name, client_ip,
       COUNT(*) AS failed_attempts,
       MIN(event_timestamp) AS first_attempt,
       MAX(event_timestamp) AS last_attempt
FROM SNOWFLAKE.ACCOUNT_USAGE.LOGIN_HISTORY
WHERE event_timestamp >= DATEADD('day', -7, CURRENT_TIMESTAMP())
  AND is_success = 'NO'
GROUP BY user_name, client_ip
HAVING COUNT(*) > 5
ORDER BY failed_attempts DESC;

-- Login activity by client type
SELECT reported_client_type,
       COUNT(*) AS login_count,
       COUNT(CASE WHEN is_success = 'YES' THEN 1 END) AS successful,
       COUNT(CASE WHEN is_success = 'NO' THEN 1 END) AS failed
FROM SNOWFLAKE.ACCOUNT_USAGE.LOGIN_HISTORY
WHERE event_timestamp >= DATEADD('day', -30, CURRENT_TIMESTAMP())
GROUP BY reported_client_type
ORDER BY login_count DESC;

-- Users logging in from multiple countries (distinct IPs)
SELECT user_name, COUNT(DISTINCT client_ip) AS distinct_ips
FROM SNOWFLAKE.ACCOUNT_USAGE.LOGIN_HISTORY
WHERE event_timestamp >= DATEADD('day', -7, CURRENT_TIMESTAMP())
  AND is_success = 'YES'
GROUP BY user_name
HAVING COUNT(DISTINCT client_ip) > 3
ORDER BY distinct_ips DESC;
```

## SESSIONS

**Purpose:** Records information about user sessions, including session duration, client information, and authentication details.

**Data Latency:** 3 hours  
**Retention:** 1 year

**Key Columns:**

| Column | Type | Description |
|--------|------|-------------|
| `SESSION_ID` | NUMBER | Unique session identifier |
| `CREATED_ON` | TIMESTAMP_LTZ | When the session was created |
| `USER_NAME` | TEXT | User associated with the session |
| `AUTHENTICATION_METHOD` | TEXT | Method used to authenticate |
| `LOGIN_EVENT_ID` | NUMBER | ID from LOGIN_HISTORY |
| `CLIENT_APPLICATION_ID` | TEXT | Identifier for the client application |
| `CLIENT_APPLICATION_VERSION` | TEXT | Version of the client application |
| `CLIENT_ENVIRONMENT` | VARIANT | JSON with client environment details |
| `CLOSED_ON` | TIMESTAMP_LTZ | When the session was closed |

**Sample Query:**
```sql
-- Find long-running sessions
SELECT session_id, user_name, created_on, closed_on,
       DATEDIFF('minute', created_on, COALESCE(closed_on, CURRENT_TIMESTAMP())) AS duration_minutes
FROM SNOWFLAKE.ACCOUNT_USAGE.SESSIONS
WHERE created_on >= DATEADD('day', -7, CURRENT_TIMESTAMP())
ORDER BY duration_minutes DESC
LIMIT 20;
```

## QUERY_HISTORY_BY_* Convenience Functions

Snowflake also provides table functions in `INFORMATION_SCHEMA` for lower-latency, targeted queries:

```sql
-- Last hour of queries (near real-time via INFORMATION_SCHEMA)
SELECT *
FROM TABLE(INFORMATION_SCHEMA.QUERY_HISTORY_BY_USER(
  USER_NAME => CURRENT_USER(),
  RESULT_LIMIT => 100
));

-- Queries on a specific warehouse in the last 7 days
SELECT *
FROM TABLE(INFORMATION_SCHEMA.QUERY_HISTORY_BY_WAREHOUSE(
  WAREHOUSE_NAME => 'MY_WAREHOUSE',
  END_TIME_RANGE_START => DATEADD('day', -7, CURRENT_TIMESTAMP())
));
```

## Dashboard Use Cases

### 1. Query Performance Overview
```sql
-- Hourly query counts and P95 latency
SELECT DATE_TRUNC('hour', start_time) AS hour,
       COUNT(*) AS query_count,
       PERCENTILE_CONT(0.50) WITHIN GROUP (ORDER BY total_elapsed_time) / 1000 AS p50_sec,
       PERCENTILE_CONT(0.95) WITHIN GROUP (ORDER BY total_elapsed_time) / 1000 AS p95_sec,
       PERCENTILE_CONT(0.99) WITHIN GROUP (ORDER BY total_elapsed_time) / 1000 AS p99_sec
FROM SNOWFLAKE.ACCOUNT_USAGE.QUERY_HISTORY
WHERE start_time >= DATEADD('day', -7, CURRENT_TIMESTAMP())
GROUP BY 1
ORDER BY 1 DESC;
```

### 2. Security Anomaly Detection
```sql
-- Off-hours logins (outside 6am–10pm local time)
SELECT user_name, event_timestamp, client_ip, reported_client_type
FROM SNOWFLAKE.ACCOUNT_USAGE.LOGIN_HISTORY
WHERE event_timestamp >= DATEADD('day', -7, CURRENT_TIMESTAMP())
  AND is_success = 'YES'
  AND (EXTRACT(HOUR FROM CONVERT_TIMEZONE('America/Los_Angeles', event_timestamp)) < 6
       OR EXTRACT(HOUR FROM CONVERT_TIMEZONE('America/Los_Angeles', event_timestamp)) >= 22)
ORDER BY event_timestamp DESC;
```
